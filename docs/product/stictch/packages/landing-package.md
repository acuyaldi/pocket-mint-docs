# Product Identity

Pocket Mint is a **Private Financial Workspace**.

Pocket Mint helps users understand:

- what they own
- what they owe
- what requires attention
- what happened recently

Pocket Mint is not:

- an expense tracker
- a banking application
- a budgeting game
- a crypto dashboard
- an analytics platform

The Landing page should immediately communicate this identity.

The experience should feel:

- calm
- trustworthy
- modern
- professional
- premium
- privacy-first

Never playful.

Never marketing-heavy.

Never aggressive.

Screen Name

Landing Page

# Screen Goal

The Landing page should convince visitors through clarity rather than persuasion.

Visitors should understand:

What Pocket Mint is.

Why it exists.

How it helps.

Why they can trust it.

How to get started.

# Layout Rules

Desktop

- 40px safe area
- generous whitespace
- readable content width
- balanced sections

Tablet

- reduced columns
- comfortable reading width

Mobile

- single-column narrative
- sequential reading
- no horizontal scrolling

# Composition Rules

The page communicates

Identity

↓

Trust

↓

Understanding

↓

Confidence

↓

Action

Every section should answer the next visitor question.

Whitespace should improve reading.

Never create decorative empty space.

# Information Density

Landing follows the lightest information density in Pocket Mint.

Whitespace communicates confidence.

Not emptiness.

Avoid overcrowding.
