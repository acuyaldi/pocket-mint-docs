---

# Purpose

Describe how this screen should be visually composed.

This document defines:

- visual hierarchy
- reading flow
- composition
- visual emphasis
- page balance

It does not define:

- business logic
- spacing values
- typography scale
- colors

Those belong to other design documents.

---

# Composition Philosophy

Describe how users should perceive the page.

The composition should support the purpose of the screen.

Every composition decision should reduce cognitive effort.

Users should understand the page before consciously exploring it.

---

# Focal Point

Define the primary visual anchor.

Only one focal point should exist.

Every supporting section should reinforce the focal point.

Never compete with it.

---

# Composition Strategy

Describe how the page should feel.

Examples

Balanced

Focused

Calm

Structured

Guided

Professional

Avoid emotional or decorative language.

---

# Desktop Composition

Describe how information should be arranged.

Examples

Single dominant hero

Supporting grid

Balanced columns

Wide reading area

Avoid filling available width simply because it exists.

---

# Tablet Composition

Describe tablet adaptation.

Reduce simultaneous information.

Preserve hierarchy.

Increase readability.

Avoid desktop density.

---

# Mobile Composition

Describe mobile composition.

One primary section at a time.

Natural vertical reading.

Comfortable scrolling.

No horizontal scrolling.

Mobile should feel guided.

Not compressed.

---

# Content Balance

Describe visual balance.

Balance should be achieved through:

- hierarchy
- grouping
- composition

Never decorative elements.

Avoid visually empty regions.

Avoid visually overloaded regions.

---

# Empty Space Strategy

Whitespace should improve understanding.

Increase whitespace around major regions.

Maintain compact grouping inside related information.

Whitespace should support hierarchy.

Not replace it.

# Emphasis Rules

Visual emphasis should come from:

- composition
- hierarchy
- grouping
- proportion

Avoid relying on:

- oversized typography
- saturated colors
- decorative illustrations
- excessive shadows

Information should remain the focus.

---

# Responsive Consistency

Desktop

Tablet

Mobile

should communicate the same story.

Only layout changes.

Hierarchy does not.

Purpose does not.

---

# Anti Patterns

Do not:

- create multiple focal points
- create competing heroes
- overload the page
- leave excessive empty regions
- overuse cards
- use decorative whitespace
- break reading flow
- increase visual noise

Composition should always support clarity.

---

# Acceptance Criteria

The composition is successful when:

✓ Users immediately understand where to look.

✓ Visual hierarchy matches business importance.

✓ Reading flow feels natural.

✓ Primary information dominates.

✓ Supporting information never competes.

✓ Desktop feels balanced.

✓ Mobile feels guided.

✓ The screen feels calm.

✓ The screen feels intentional.

---

# Definition of Done

A composition document is complete when:

✓ Reading flow is documented.

✓ Visual hierarchy is fixed.

✓ Focal point is defined.

✓ Visual weight is assigned.

✓ Responsive composition is documented.

✓ Progressive disclosure is defined.

✓ Anti-patterns are documented.

✓ Acceptance criteria are measurable.

This document becomes the visual composition contract for the screen.
