# Pocket Mint Design Review Checklist

Version: 1.0

Status: Foundation

Owner: Pocket Mint

Category: Design QA

---

# Purpose

This document provides a standardized checklist for reviewing every UI generated in Google Stitch.

It ensures every screen maintains Pocket Mint's identity, usability, consistency, and product quality.

Every generated screen must pass this review before implementation.

---

# Review Process

Each screen should be reviewed in the following order:

1. Product Identity

2. Information Hierarchy

3. Layout & Composition

4. Visual Language

5. Interaction

6. Responsive Design

7. Accessibility

8. Content

9. Overall Quality

Do not review visual appearance before reviewing information hierarchy.

# Definition of Done

A design is approved only when:

✓ Product identity is preserved.

✓ Information hierarchy is correct.

✓ Composition is balanced.

✓ Visual language is consistent.

✓ Responsive behavior follows the design system.

✓ Accessibility is maintained.

✓ Content follows Pocket Mint guidelines.

✓ The screen is implementation-ready.

This checklist must be completed before any Pocket Mint screen is frozen for frontend implementation.
