# Pocket Mint Design Principles

> Pocket Mint v2
> Version: 1.0
> Status: Draft

---

# Philosophy

Pocket Mint is a financial workspace.

It is not a budgeting game.

It is not a banking application.

It is not a social product.

Every screen should help users understand their financial position with confidence.

---

# Core Experience

Users should feel:

- Calm
- Focused
- In Control
- Trustworthy
- Organized

Never overwhelmed.

Never distracted.

Never manipulated.

---

# Design Values

## Financial Clarity

Information always comes before decoration.

---

## Trust

Financial software should look stable.

Avoid playful interactions.

Avoid exaggerated animations.

---

## Progressive Disclosure

Show important information first.

Reveal detail only when needed.

---

## Consistency

Identical concepts must always look identical.

Wallets always look like wallets.

Debt always looks like debt.

Transactions always look like transactions.

---

## Explainability

Every number should have an obvious meaning.

Users should never wonder:

"What does this number mean?"

---

## Visual Hierarchy

Hierarchy is created through:

- size
- spacing
- grouping

Not through excessive colors.

---

## Calm Interface

Prefer whitespace over borders.

Prefer typography over decoration.

Prefer structure over effects.

---

# What Pocket Mint Is NOT

Do not imitate:

- crypto dashboards
- trading terminals
- banking apps
- budgeting games

Pocket Mint should resemble a professional financial workspace.

---

# Financial Design Rules

Net Worth is always the primary financial metric.

Assets are never visually confused with Debt.

Debt is never minimized.

Negative financial information should remain visible.

---

# Layout Principles

Every screen answers one primary question.

Every section has one purpose.

No duplicated information.

No decorative charts.

---

# Interaction Principles

Primary actions are always visible.

Dangerous actions require confirmation.

Financial actions never surprise users.

---

# Motion

Motion should communicate state.

Never exist purely for decoration.

---

# Accessibility

Readable typography.

Strong contrast.

Keyboard friendly.

Color is never the only indicator.

Tabular numbers for all currency.

---

# Stitch Notes

Prioritize:

- clarity
- hierarchy
- professionalism
- whitespace
- financial readability

Do not optimize for visual novelty.

Optimize for financial confidence.
