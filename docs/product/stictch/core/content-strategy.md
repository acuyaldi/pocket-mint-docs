# Pocket Mint Content Strategy

Version: 1.0

Status: Foundation

Owner: Pocket Mint

Category: Content Strategy

---

# Purpose

This document defines how information should be presented throughout Pocket Mint.

Content strategy ensures every screen communicates only the information users need at the appropriate moment.

Pocket Mint prioritizes understanding over information quantity.

Every piece of content should reduce uncertainty.

Not increase it.

---

# Content Philosophy

Pocket Mint is a financial workspace.

Content exists to support financial awareness.

Not engagement.

Not entertainment.

Not marketing.

Every sentence, label, and value should have a purpose.

If content does not improve financial understanding, it should not exist.

---

# Core Principles

Content should be:

Clear

↓

Relevant

↓

Actionable

↓

Minimal

↓

Consistent

Every screen should communicate only what users need for the current task.

---

# Writing Tone

Pocket Mint communicates professionally.

Friendly.

Calm.

Confident.

Never casual.

Never playful.

Never promotional.

Avoid emotional language.

Avoid exaggerated claims.

---

# Language Style

Use simple language.

Avoid financial jargon whenever possible.

When financial terminology is necessary, prioritize common understanding.

Users should never require financial expertise to understand the interface.

---

# Labels

Labels should be:

Short

Specific

Consistent

Avoid synonyms for the same concept.

Example

Always use:

Dashboard

Dompet

Transaksi

Cicilan

Akun

Do not alternate terminology.

---

# Numbers

Numbers are the most important content.

Financial values should always receive the highest emphasis.

Supporting text should never compete with financial values.

Numbers should remain easy to scan.

---

# Dates

Dates should communicate context.

Examples

Today

Yesterday

Tomorrow

or

12 Jul 2026

Use whichever format improves understanding.

Avoid unnecessary timestamp precision.

---

# Currency

Always display currency consistently.

Avoid mixing formats.

Currency should remain recognizable before users read the value.

Financial consistency builds trust.

---

# Dashboard Content

Dashboard communicates awareness.

Content should answer:

Where am I financially?

What needs attention?

What happened recently?

Dashboard should summarize.

Never explain excessively.

---

# Wallet Content

Wallets communicate ownership.

Content should answer:

What do I own?

Where is it stored?

How much is available?

Wallets should prioritize management.

Not reporting.

---

# Transaction Content

Transactions communicate history.

Content should answer:

What happened?

When?

Where?

How much?

Transaction screens may contain more detail than Dashboard.

---

# Installment Content

Installments communicate obligations.

Content should answer:

What do I owe?

When is payment due?

How much remains?

Installments should prioritize clarity over visual richness.

---

# Landing Content

Landing introduces Pocket Mint.

Landing should communicate:

Product identity.

Core value.

Privacy.

Trust.

Landing is the only screen where persuasive copy is appropriate.

---

# Supporting Content

Supporting information should explain.

Not distract.

Examples

Wallet Type

Institution

Category

Description

Reference

Supporting information remains visually secondary.

---

# Empty States

Empty states should reassure users.

Never shame users.

Good

"No transactions recorded yet."

Good

"Create your first wallet."

Avoid

"You haven't done anything."

Avoid

"Your account is empty."

Empty states should encourage progress.

Not highlight absence.

---

# Error Messages

Error messages should answer:

What happened?

What can the user do?

Avoid technical terminology.

Avoid blame.

Good

"Unable to load transactions. Please try again."

Avoid

"Unknown exception."

Avoid

"500 Internal Server Error."

---

# Success Messages

Success messages should be brief.

Examples

Transaction added.

Wallet updated.

Installment marked as paid.

Avoid unnecessary celebration.

Financial software should remain calm.

---

# Warning Messages

Warnings should communicate urgency without creating anxiety.

Good

"Payment due tomorrow."

Avoid

"Urgent! Pay immediately!"

Warnings should encourage action.

Not panic.

---

# Notification Strategy

Only notify users when meaningful action is required.

Examples

Upcoming payment.

Failed synchronization.

Successful transaction.

Avoid unnecessary notifications.

Silence is preferable to noise.

---

# Progressive Disclosure

Show only the information necessary for the current context.

Overview

↓

Summary

↓

Details

↓

Management

↓

History

Avoid exposing deep information immediately.

---

# Content Density

Pocket Mint follows balanced density.

Dashboard

Light

Wallets

Medium

Transactions

High

Installments

Medium

Landing

Light

Density should match user intent.

---

# Scanability

Users should understand every screen through quick scanning.

Use:

Headings

Grouping

Whitespace

Financial values

Avoid:

Long paragraphs

Dense explanations

Wall-of-text layouts

---

# Consistency

The same information should always use the same wording.

The same action should always use the same label.

Consistency reduces cognitive effort.

---

# Accessibility

Content should remain understandable.

Avoid:

Abbreviations without explanation.

Complex terminology.

Long instructions.

Sentences should remain concise.

---

# Privacy

Content should avoid exposing sensitive information unnecessarily.

Future features may support:

Hidden balances

Masked values

Privacy Mode

Content should accommodate these scenarios.

---

# Anti Patterns

Do not:

Use marketing copy inside authenticated screens.

Celebrate ordinary financial actions.

Overexplain obvious concepts.

Duplicate information.

Use inconsistent terminology.

Display fake financial insights.

Prioritize decoration over understanding.

---

# Acceptance Criteria

Content strategy is successful when:

✓ Users immediately understand the purpose of each screen.

✓ Financial information is always prioritized.

✓ Labels remain consistent.

✓ Messages remain calm.

✓ Error messages are actionable.

✓ Empty states encourage progress.

✓ Dashboard summarizes.

✓ Detail pages manage.

---

# Definition of Done

The content strategy is complete when every Pocket Mint screen communicates information consistently, professionally, and with minimal cognitive effort.

This document serves as the authoritative content reference for all Pocket Mint interfaces.
