# Purpose

The Landing page introduces Pocket Mint to new users.

Unlike authenticated screens, the Landing page communicates identity before functionality.

Its purpose is to build trust.

Not excitement.

---

# Product Role

The Landing page acts as the product's public entrance.

It should communicate:

- What Pocket Mint is.
- Why it exists.
- Why users should trust it.
- How to begin.

---

# User Goals

Visitors should quickly understand:

- What Pocket Mint does.
- Who it is for.
- Why it is different.
- Whether it fits their needs.
- How to start using it.

---

# Primary User Questions

The Landing page should answer:

What is Pocket Mint?

Why should I use it?

How is it different?

Is my financial data private?

How do I get started?

# Business Rules

Landing should never expose user financial information.

Landing should never imitate a banking homepage.

Marketing language should remain restrained.

Privacy should remain visible.

# Things To Avoid

Do not:

Use fintech hype.

Display fake dashboards.

Use exaggerated statistics.

Use fake testimonials.

Overload the page.

Compete with the authenticated experience.
